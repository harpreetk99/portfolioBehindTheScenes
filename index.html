Creating Your Portfolio

Creating Simple Web Portfolio using simple HTML, JavaScript, and CSS and publish it on Github Pages.

Portfolio Creation

In this post, I will guide you though the process of quickly creating your portfolio and set it up on Github Pages. This is a good start if you are at the point where you have some projects to show (on Github or somewhere else) but you don’t have a personal portfolio website to show them on.

Decisions to Make

Alright, there are many decisions you should make before starting. For example, how should page items be stored? Should they be stored in their own folder or just as .md file? Where do you store images? And a lot more.

But since this project’s scope is not very large, I’m going to go with a very simple file tree. I'm choosing not to store items in JSON objects. Other resources (like images) can be stored in /imgs/.

These decisions were made to make file reading and writing simpler for this specific scenario, but you can always change them to whatever is better suited for your project.

Folder structure
|-- app
|---- img
|---- pages
|---- data
|---- js
|---- pages
|---- css
|--- index.html

Let’s start from the top (you may now clone the boilerplate to follow the structure and the code in detail as we go through them):

app/img Contains the image files.

app/pages Contains the html templates. The structure under pages defines the url structure of our website including the nested routes, so pages/products will result in www.[your-domain].com/products and pages/products/latest will result in www.[your-domain].com/products/latest

app/css Contains the CSS files. The structure under this must be identical to the folder structure of the html templates but the existence of the styles are not required if a certain page does not have any styles.

app/scripts Contains the javascript files.

app/data contains the data object you wish to pass to the html template.

import aPages from "../pages/index.js";  
class Page {
   constructor() {
    this.sName = "Harpreet Kaur";
   }
   render() {}
}

class Header extends Page {
  render() {}
}

class Section extends Page {
  constructor() {}
  render() {}
}

class Article extends Page {
  render() {
    for (let n = 0; n < aPages.length; n++) {
      $("article#pages").append(`<section id="${aPages[n].title}"></section>`);
      new Section(aPages[n]).render();
    }
  }
}

class Footer extends Page {
  render() {}
}
index.js

Alright, so if you look at the architectural code below, you’ll notice that the Header, Article and Footer component is encapsulated by the Portfolio layout. This means that when we load the Portfolio layout, it will also load the all three components. 

class Portfolio extends Page {
constructor() {
  super();
  this.header = new Header();
  this.article = new Article();
  this.footer = new Footer();
}
render() {
  this.header.render();
  this.article.render();
  this.footer.render();
}
}

$(document).ready(() => {
  new Portfolio().render();
});

data/projects.js

We will import JSON array from data/projects.js. Now that our state has an array of data to work with, we can loop over it using for loop by modifying our render() and pass object of array as a parameter to Project class constructor. We will call render method of Project class to render each project. Each iteration will build new Project markup : 

import projectsData from "../data/projects.js";

class Section extends Page {
constructor(oOptions) {
  super();
  this.oOptions = oOptions;
}

render() {
  if (this.oOptions.title == "Projects") {
    $(`#${this.oOptions.title}`).append(`<div class="projects-wrapper">
    <h1 class="projectPageTitle">Projects.</h1>
    <div class="projects-container"></div>
    </div>`);
    for (let n = 0; n < projectsData.length; n++) {
      $(".projects-container").append(
        `<div id="${projectsData[n].title
          .replace(/s+/g, "-")
          .toLowerCase()}"></div>`
      );
      new Project(projectsData[n]).render();
    }
  }
}
}

Create Project class and add a constructor that takes in the object. Save the object in internal variables (this scope) in the constructor function. When we are finished creating the markup, we simply use the jQuery .append() method to insert our markup into the DOM, using the ID of a specific DIV element.

class Project extends Page {
constructor(pOptions) {
super();
this.pOptions = pOptions;
}

render() {
const className = this.pOptions.title.replace(/s+/g, "-").toLowerCase();

const codeLink = "#";

if (this.pOptions.codeLink) {
  codeLink = this.pOptions.codeLink;

const techList = this.pOptions.techList.join(", ");

Hosting the Portfolio

Now that our project is finished we can host it on GitHub Pages. If you do not have a GitHub account, it’s free to create one. GitHub offers an option to host your portfolio free of charge while still being accessible to a large audience.

Github Repository of Portfolio Project

Steps to host your portfolio on Github

This section will take you through the steps to publish your website on Github Pages. Once the account is created you’ll want to go to the profile page. Next, select “Repositories” and choose “Create a New Repository”. To use GitHub Pages you need to name the project similar to the one I have shown. First, input your GitHub profile name and keep the rest. Now press “Create Repository”. 

[yourusername].github.io

You do not need to know how to use Git through the command line since everything can be done through the website. Select “Upload Existing File” and copy all the files from your project to the website. You can copy the whole folder since the index file has to be easily accessible. Once all of the files have been uploaded, the portfolio is officially ready. Your portfolio is published as the project name. Copy the website name we’ve used above and you can preview the portfolio!
